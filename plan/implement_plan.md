# PM-Skills 产品经理技能包-superPM 全量落地实施方案
## （颗粒度到单技能·全生命周期闭环·100%对标gstack和superpower工业级标准）
本方案完全遵循 **Claude Code 官方 Custom Skills 开发规范**，对标gstack、superpowers 的skills技能包 底层实现逻辑，覆盖「技能底层架构→单技能详细实现→全生命周期联动引擎→触发机制→工程化落地」全链路，颗粒度细化到每个技能的子功能、输入输出规范、联动依赖、触发条件、异常处理，可直接用于开发落地。

## 一、方案总览与底层架构设计
### 1. 核心定位与适配规范
- **定位**：专为产品经理打造的Claude Code本地技能包，覆盖产品全生命周期，内置Web联网搜索能力，融合全行业产品方法论，实现「一人=全产品团队」的效率提效。
- **适配规范**：100%对齐Claude Code Custom Skills 官方标准，与gstack安装、使用、执行逻辑完全一致，无学习成本。
- **运行环境**：Claude Code ≥ 0.15.0、Node.js ≥ 18 LTS、Git、macOS/Linux/WSL（Windows原生CMD不兼容）。

### 2. 底层6层架构设计
| 架构层 | 核心职责 | 技术实现 | 对接规范 |
|--------|----------|----------|----------|
| 适配层 | 对接Claude Code斜杠命令系统，处理输入输出 | Shell/Node.js脚本、Claude Code Skills API | 固定目录`~/.claude/skills/pm-skills`，`skill.json`配置文件 |
| 核心引擎层 | 状态机、规则引擎、数据总线、联动逻辑、触发校验 | Node.js核心服务、JSON Schema校验、状态机管理 | 全技能统一入口，统一异常处理 |
| 技能层 | 43个独立技能的封装实现，每个技能独立可插拔 | 每个技能对应独立脚本，标准化输入输出 | 统一的函数签名、参数规范、返回格式 |
| 搜索层 | 联网搜索能力封装，数据抓取、清洗、结构化、过滤 | 对接Claude Code内置联网API/SerpAPI，自定义数据源配置 | 统一搜索接口，全技能可调用 |
| 存储层 | 项目配置管理、数据总线存储、执行日志、输出文档 | 项目根目录`PM-CLAUDE.md`、本地缓存目录`~/.claude/cache/pm-skills` | Markdown+结构化JSON双格式，人机可读 |
| 异常层 | 全局异常捕获、错误提示、重试机制、日志上报 | 全局异常中间件、分级错误码、自动重试逻辑 | 统一错误格式，用户友好提示 |

## 二、核心引擎与触发机制详细实现
### 1. 全局状态机设计（全生命周期流转核心）
项目状态是所有技能触发、联动的核心依据，状态流转不可逆（迭代循环除外），严格对应产品经理真实工作流。

| 状态编码 | 状态名称 | 触发条件 | 可执行技能 | 流转目标 |
|----------|----------|----------|------------|----------|
| S0 | 未初始化 | 项目新建，未执行任何核心技能 | `/pm-demand`（强制入口）、`/pm-pool`（成熟项目入口） | 执行`/pm-demand`→S1；执行`/pm-pool`→S1 |
| S1 | 需求调研完成 | 已执行`/pm-demand`或`/pm-pool`，数据总线已写入核心需求数据 | 模块1所有技能、模块2核心技能（`/pm-brd`/`/pm-mrd`/`/pm-prd`） | 完成MVP方案制定→S2 |
| S2 | 方案设计完成 | 已执行`/pm-mvp`+`/pm-prd`，数据总线已写入完整产品方案 | 模块2所有技能、模块4核心技能（`/pm-agile`/`/pm-cross`/`/pm-risk`） | 完成上线准备→S3 |
| S3 | 上线准备完成 | 已执行`/pm-risk`+`/pm-search-legal`+`/pm-release`，数据总线已写入上线方案 | 模块4所有技能、模块3核心技能（`/pm-aarrr`/`/pm-growth`） | 版本上线→S4 |
| S4 | 上线迭代中 | 已执行版本上线，数据总线已写入上线结果 | 模块3所有技能、模块4复盘相关技能 | 完成版本复盘→S1（开启下一轮迭代） |

### 2. 4类触发机制 详细实现规则
#### （1）强制触发机制（锁级规则，不可绕过）
- **唯一触发入口**：仅`/pm-demand`（0-1新产品）、`/pm-pool`（成熟产品迭代）
- **实现逻辑**：
  1.  项目状态为S0时，所有核心技能的执行请求都会被拦截，返回强制引导：「请先执行`/pm-demand`完成需求调研，或执行`/pm-pool`导入需求池，解锁后续技能」
  2.  仅当`/pm-demand`或`/pm-pool`执行成功，项目状态更新为S1，才会解锁其他核心技能
  3.  上线强制校验：执行`/pm-release`前，必须已执行`/pm-search-legal`合规检查，否则拦截，返回：「请先执行`/pm-search-legal`完成合规风险排查，再制定上线方案」

#### （2）自动触发机制（规则引擎驱动，新手模式默认开启）
- **触发条件**：`PM-CLAUDE.md`中`auto_trigger: true`且`user_mode: beginner`
- **实现逻辑**：每个技能都定义了「后置自动触发规则」，当前序技能执行成功（返回码0），且规则匹配时，自动执行后置技能，无需用户手动输入。
- **全局自动触发规则表**：
  | 前置技能 | 自动触发的后置技能 | 触发条件 |
  |----------|----------------------|----------|
  | `/pm-demand` | `/pm-search-market` → `/pm-verify` | 执行成功，无参数错误 |
  | `/pm-verify` | `/pm-value` → `/pm-kano` → `/pm-rice` | 执行成功，核心需求不为空 |
  | `/pm-rice` | `/pm-mvp` | 执行成功，优先级清单不为空 |
  | `/pm-mvp` | `/pm-brd` → `/pm-mrd` → `/pm-prd` | 执行成功，MVP功能清单不为空 |
  | `/pm-prd` | `/pm-proto` → `/pm-ui` → `/pm-tech` → `/pm-data` | 执行成功，PRD内容完整 |
  | `/pm-tech` | `/pm-cross` → `/pm-risk` | 执行成功，技术排期不为空 |
  | `/pm-risk` | `/pm-search-legal` | 执行成功，风险清单不为空 |
  | `/pm-search-legal` | `/pm-release` | 执行成功，无致命合规风险 |
  | `/pm-release` | `/pm-aarrr` → `/pm-growth` | 版本上线完成，执行成功 |
  | `/pm-data-report` | `/pm-iteration` | 执行成功，迭代需求不为空 |
  | `/pm-iteration` | `/pm-demand-review` | 执行成功，迭代计划不为空 |
  | `/pm-retro` | `/pm-long` | 执行成功，复盘内容完整 |

#### （3）条件触发机制（参数校验引擎驱动，不可空跑）
- **实现逻辑**：每个技能都定义了「必填前置参数」，执行前会自动校验数据总线中是否存在对应参数，参数不完整则拦截执行，返回明确的补充提示。
- **参数校验规则**：
  1.  必填参数必须存在，且非空
  2.  参数格式必须符合JSON Schema定义
  3.  专家模式下，可通过手动输入参数覆盖前置依赖，灵活执行
- **核心技能必填前置参数表**（示例）：
  | 技能命令 | 必填前置参数 | 对应前置技能 |
  |----------|--------------|--------------|
  | `/pm-verify` | `demand_report` | `/pm-demand` |
  | `/pm-mvp` | `rice_priority`、`core_value_tree` | `/pm-rice`、`/pm-value` |
  | `/pm-prd` | `mvp_plan`、`demand_report` | `/pm-mvp`、`/pm-demand` |
  | `/pm-abtest` | `iteration_plan`、`optimization_target` | `/pm-iteration` |
  | `/pm-release` | `legal_compliance_report`、`risk_control_plan` | `/pm-search-legal`、`/pm-risk` |
  | `/pm-iteration` | `data_report`、`user_feedback_analysis` | `/pm-data-report`、`/pm-feedback` |

#### （4）手动触发机制（灵活无限制，适配专项场景）
- **实现逻辑**：无强制前置依赖，可随时执行，支持读取数据总线现有数据，也支持用户手动输入完整参数，适配突发工作、专项分析、汇报等场景。
- **手动触发技能清单**：`/pm-brain`、`/pm-journey`、`/pm-moscow`、`/pm-search-comp`、`/pm-feature`、`/pm-user-level`、`/pm-commercial`、`/pm-funnel`、`/pm-search-data`、`/pm-search-opinion`、`/pm-change`、`/pm-report`、`/pm-startup`

### 3. 联网搜索能力 全场景实现规范
所有带🟢标记的搜索技能，统一调用搜索层接口，遵循以下标准化实现规则：
1.  **数据源优先级**：用户自定义搜索源 > 官方权威数据源 > 全网通用搜索
2.  **数据过滤规则**：默认仅保留1年内的有效数据，过滤无效、重复、非权威信息
3.  **Query生成规则**：基于用户输入的关键词+行业赛道+产品定位，自动生成3-5组精准搜索Query，覆盖核心需求
4.  **数据结构化规则**：搜索结果自动清洗、去重、分类，转化为结构化JSON数据，写入数据总线，同时嵌入到技能输出的文档中
5.  **异常处理**：搜索失败时，自动重试2次，仍失败则提示用户检查网络权限，同时输出无搜索数据的基础版本结果

## 三、43个技能 全量详细实现（颗粒度到子功能）
### 模块1：需求洞察与价值验证（强制入口模块·11项）
**模块定位**：产品工作的起点，解决「做正确的事」的核心问题，唯一强制入口模块，所有后续流程均基于本模块输出展开。
**模块联动规则**：本模块所有技能的输出，自动写入数据总线，作为模块2、3、4的核心输入；新手模式下，技能按顺序自动触发。

### 3. 一键安装脚本`setup`
```bash
#!/bin/bash
set -e

echo "正在安装 PM-Skills 产品经理技能包 v1.0.0..."

# 检查Node.js版本
if ! command -v node &> /dev/null; then
  echo "错误：未安装Node.js，请先安装Node.js ≥18 LTS"
  exit 1
fi

NODE_VERSION=$(node -v | cut -d'v' -f2 | cut -d'.' -f1)
if [ "$NODE_VERSION" -lt 18 ]; then
  echo "错误：Node.js版本过低，请升级到Node.js ≥18 LTS"
  exit 1
fi

# 安装依赖
npm install

# 给所有技能脚本添加执行权限
chmod +x ./bin/*

# 检查Claude Code技能目录
SKILLS_DIR="$HOME/.claude/skills"
if [ ! -d "$SKILLS_DIR" ]; then
  mkdir -p "$SKILLS_DIR"
fi

echo "✅ superPM Skills 安装完成！共加载43个技能"
echo "📖 请在项目根目录创建 PM-CLAUDE.md 配置文件，即可开始使用"
echo "🚀 第一个命令请执行 /pm-demand 开启你的产品项目"
```

## 六、落地保障与异常处理规范
1.  **权限保障**：所有脚本仅在用户授权的目录内执行，无越权操作，完全符合Claude Code安全规范
2.  **数据安全**：所有项目数据仅存储在用户本地项目目录，无数据上传、无隐私泄露

5.  **更新保障**：内置`/superpm-upgrade`命令，一键升级到最新版本，无需手动重新安装
