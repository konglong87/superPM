## 二、用户模块功能拆解

### 2.1 用户注册功能

#### 基本信息

**功能名称**：用户注册

**功能描述**：允许新用户通过手机号/邮箱注册账号

**功能目标**：让用户快速创建账号，开始使用产品

**优先级**：P0（核心功能）

**用户价值**：获得个人账号，享受个性化服务

---

#### 用户场景分析

**典型用户**：
- 25-35岁，一线城市
- 习惯使用互联网产品
- 对便捷性要求高

**使用动机**：
- 需要购买商品
- 需要收藏商品
- 需要查看订单

**使用场景**：
- 首次使用产品
- 促销活动吸引
- 朋友推荐

**使用频率**：低频（每个用户只注册一次）

---

#### 功能流程

**前置条件**：
- 用户未登录
- 用户同意用户协议

**主流程**：

```
1. 用户点击"注册"按钮
   ↓
2. 进入注册页面
   ↓
3. 用户输入手机号/邮箱
   ↓
4. 系统发送验证码
   ↓
5. 用户输入验证码
   ↓
6. 系统验证验证码
   ↓
7. 用户设置密码
   ↓
8. 系统验证密码强度
   ↓
9. 用户点击"完成注册"
   ↓
10. 系统创建账号
   ↓
11. 自动登录并跳转首页
```

**后置操作**：
- 发送欢迎短信/邮件
- 创建用户默认数据（如购物车）
- 记录注册来源（用于数据统计）

---

#### 分支流程

**分支流程1：手机号已注册**

- **触发条件**：用户输入的手机号已存在
- **系统处理**：
  - 后端检查手机号是否存在
  - 返回"手机号已注册"错误
- **用户反馈**：
  - 实时提示"该手机号已注册，请直接登录"
  - 提供"去登录"按钮
- **后续操作**：点击"去登录"跳转登录页

**分支流程2：验证码错误**

- **触发条件**：用户输入的验证码不正确或已过期
- **系统处理**：
  - 验证验证码正确性
  - 返回错误信息
- **用户反馈**：
  - 提示"验证码错误"或"验证码已过期"
  - 提供"重新发送"按钮
- **后续操作**：用户可重新获取验证码

**分支流程3：密码强度不足**

- **触发条件**：用户设置的密码不符合安全要求
- **系统处理**：
  - 实时检查密码强度
  - 提示强度不足原因
- **用户反馈**：
  - 密码强度指示器（弱/中/强）
  - 提示：密码至少8位，包含数字和字母
- **后续操作**：用户修改密码直到符合要求

---

#### 界面元素

**页面结构**：

**顶部**：
- Logo
- 标题："创建账号"

**主体**：
- 手机号/邮箱输入框
  - 类型：文本输入
  - 校验：格式校验，实时提示
  - 占位符："请输入手机号"

- 验证码输入框
  - 类型：文本输入（6位数字）
  - 校验：长度校验
  - 占位符："请输入验证码"

- 获取验证码按钮
  - 初始状态："获取验证码"
  - 倒计时状态："60秒后重新获取"
  - 禁用状态：已发送验证码后

- 密码输入框
  - 类型：密码输入（可切换明文）
  - 校验：密码强度实时检查
  - 占位符："请设置密码（至少8位）"

- 确认密码输入框
  - 类型：密码输入
  - 校验：与密码一致性校验
  - 占位符："请再次输入密码"

- 用户协议复选框
  - 内容："我已阅读并同意《用户协议》和《隐私政策》"
  - 默认：未勾选

- 注册按钮
  - 状态：禁用（表单未填写完整）/ 可用（表单已填写完整）
  - 文案："注册"

**底部**：
- 已有账号？去登录
- 第三方登录入口（微信、QQ）

**交互细节**：
- 输入框获得焦点时显示提示文字
- 输入框失焦时实时校验并提示错误
- 密码输入时显示强度指示器
- 注册按钮点击后显示Loading状态

**状态展示**：
- **加载状态**：注册按钮显示Loading动画
- **成功状态**：Toast提示"注册成功"，自动跳转
- **失败状态**：Toast提示错误原因，停留在当前页

---

#### 数据逻辑

**输入数据**：

| 字段名 | 类型 | 必填 | 校验规则 | 说明 |
|--------|------|------|---------|------|
| phone | String | 是 | 11位手机号 | 用户手机号 |
| code | String | 是 | 6位数字 | 验证码 |
| password | String | 是 | 8-20位，包含数字和字母 | 密码 |
| agreement | Boolean | 是 | true | 是否同意协议 |

**数据校验**：

- 手机号格式：`/^1[3-9]\d{9}$/`
- 验证码格式：`/^\d{6}$/`
- 密码强度：至少8位，包含数字和字母
- 确认密码：与密码一致

**数据存储**：

- 用户表（users）：
  - id: VARCHAR(36) - 用户ID（UUID）
  - phone: VARCHAR(11) - 手机号
  - password_hash: VARCHAR(255) - 密码哈希（bcrypt加密）
  - created_at: TIMESTAMP - 创建时间
  - updated_at: TIMESTAMP - 更新时间

**数据输出**：

- 注册成功：返回用户基本信息和Token
- 注册失败：返回错误码和错误信息

---

#### 异常处理

**前端异常**：

| 异常类型 | 处理方式 | 提示文案 |
|---------|---------|---------|
| 网络异常 | 显示错误提示，提供重试按钮 | "网络连接失败，请检查网络后重试" |
| 手机号格式错误 | 实时校验，显示错误提示 | "请输入正确的手机号" |
| 验证码错误 | 显示错误提示，允许重新输入 | "验证码错误，请重新输入" |
| 密码强度不足 | 实时提示，显示强度指示器 | "密码强度不足，请设置更复杂的密码" |
| 两次密码不一致 | 实时校验，显示错误提示 | "两次输入的密码不一致" |

**后端异常**：

| 错误码 | 错误说明 | 处理方式 |
|--------|---------|---------|
| 400 | 参数错误 | 返回具体错误字段和提示 |
| 409 | 手机号已注册 | 提示用户去登录 |
| 500 | 服务器错误 | 记录日志，提示"系统繁忙" |

**边界条件**：

- 手机号长度：限制11位，超出自动截断
- 验证码长度：限制6位，超出自动截断
- 密码长度：限制20位，超出提示"密码过长"
- 防重复提交：注册按钮点击后禁用，防止重复提交

#### 技术实现要点

**前端实现**：

- 表单验证：使用表单验证库（如React Hook Form + Yup）
- 实时校验：输入框失焦时校验，实时反馈
- 密码强度检测：实时检测密码强度并显示
- 倒计时组件：验证码发送后倒计时60秒
- 防重复提交：按钮点击后禁用，请求完成后恢复

**后端实现**：

- 验证码生成：使用短信服务商API（如阿里云短信）
- 验证码存储：存储到Redis，设置5分钟过期时间
