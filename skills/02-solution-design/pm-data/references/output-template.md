## 一、指标体系概述

### 1.1 设计目标

建立完整的数据指标体系，帮助团队：
- 衡量产品成功度
- 发现产品问题
- 指导产品优化
- 支撑业务决策

### 1.2 设计原则

- **可衡量**：指标可量化，数据可采集
- **可行动**：指标变化可指导具体行动
- **可追溯**：指标数据可追溯历史趋势
- **可对比**：可与行业基准或竞品对比

## 二、北极星指标

### 2.1 指标定义

**北极星指标**：GMV（成交总额）

**定义**：统计周期内所有订单的金额总和

**计算公式**：
```
GMV = SUM(order_amount)
```

**统计周期**：月度

**选择理由**：
- 反映电商产品的核心价值：促成交易
- 直观反映业务规模和增长
- 便于拆解为关键指标（DAU、转化率、客单价）

### 2.2 目标值

**短期目标（3个月）**：GMV达到100万
**中期目标（1年）**：GMV达到500万
**长期目标（3年）**：GMV达到5000万
**行业基准**：
- 同类电商产品平均GMV：{行业数据}
- 头部电商产品GMV：{行业数据}

## 三、关键指标体系

### 3.1 指标分层架构

```
北极星指标
    GMV（成交总额）
        ↓
关键指标（Key Metrics）
├── DAU（日活跃用户数）
├── 转化率（访问→购买）
├── 客单价（平均订单金额）
├── 复购率（重复购买用户比例）
└── 用户留存率（7日/30日）
```

### 3.2 关键指标详细定义

每个指标包含：指标定义、计算公式、数据来源、统计周期、基准值、负责人

#### 指标1：DAU（日活跃用户数）
- 定义：每天使用产品的独立用户数
- 统计口径：用户打开应用并产生至少一次有效行为
- 公式：`DAU = COUNT(DISTINCT user_id) WHERE date = today AND behavior_count > 0`
- 行业平均：{行业数据}
- 目标值：{产品目标}

#### 指标2：转化率
- 定义：访问用户中最终完成购买的用户比例
- 公式：`转化率 = COUNT(DISTINCT buyer_user_id) / COUNT(DISTINCT visitor_user_id) * 100%`
- 行业平均：2-3%
- 目标值：5%

#### 指标3：客单价
- 定义：平均每个订单的金额
- 公式：`客单价 = SUM(order_amount) / COUNT(order_id)`
- 目标值：{产品目标}

#### 指标4：复购率
- 定义：统计周期内重复购买的用户比例
- 公式：`复购率 = COUNT(重复购买用户数) / COUNT(总购买用户数) * 100%`
- 行业平均：20-30%
- 目标值：40%

#### 指标5：用户留存率
- 定义：用户在首次使用后一段时间内仍在使用的比例
- 关键留存：次日留存、7日留存、30日留存
- 行业基准：次日40-60%，7日20-40%，30日10-20%

## 四、过程指标体系

### 4.1 用户获取指标
| 指标名称 | 定义 | 基准值 |
|---------|------|--------|
| 新增用户数 | 每日新注册用户数 | 目标：100/天 |
| 获客成本（CAC） | 平均获取一个用户的成本 | < 50元 |
| 注册转化率 | 访问用户中完成注册的比例 | > 30% |

### 4.2 用户活跃指标
| 指标名称 | 定义 | 基准值 |
|---------|------|--------|
| DAU/MAU比值 | 用户粘性指标 | > 0.3 |
| 人均使用时长 | 平均每个用户的使用时长 | > 10分钟 |
| 功能使用率 | 各功能的使用用户占比 | - |

### 4.3 用户留存指标
| 指标名称 | 定义 | 基准值 |
|---------|------|--------|
| 次日留存率 | 第2天仍使用的用户比例 | > 40% |
| 7日留存率 | 第7天仍使用的用户比例 | > 20% |
| 30日留存率 | 第30天仍使用的用户比例 | > 10% |

### 4.4 用户转化指标
| 指标名称 | 定义 | 基准值 |
|---------|------|--------|

## 五、转化漏斗分析

### 5.1 核心转化漏斗

```
打开应用（100%）→ 浏览商品（70%）→ 查看详情（40%）
→ 加入购物车（20%）→ 提交订单（10%）→ 完成支付（8%）
```

**整体转化率**：8%

### 5.2 漏斗优化方向
- **当前瓶颈**：浏览→详情(40%→目标60%)、详情→加购(50%→目标70%)
- **优化建议**：优化商品列表展示，优化详情页，优化加购按钮

## 六、数据埋点方案

### 6.1 页面埋点
每个页面：page_view + page_stay 事件
- 首页：page_view_home
- 商品详情页：page_view_product_detail（含 product_id）
- 购物车页：page_view_cart
- 订单页：page_view_order

### 6.2 事件埋点
| 事件 | 触发时机 | 关键字段 |
|------|---------|---------|
| click_search | 点击搜索 | user_id, keyword, source |
| add_to_cart | 点击"加入购物车" | user_id, product_id, quantity |
| submit_order | 提交订单 | user_id, order_id, amount |
| pay_success | 完成支付 | user_id, order_id, amount, method |

### 6.3 埋点实施清单
| 页面/事件 | 埋点名称 | 优先级 |
|----------|---------|--------|
| 首页浏览 | page_view_home | P0 |
| 商品详情浏览 | page_view_product_detail | P0 |
| 点击搜索 | click_search | P0 |
| 加入购物车 | add_to_cart | P0 |
| 提交订单 | submit_order | P0 |
| 完成支付 | pay_success | P0 |

| 指标 | 告警阈值 | 告警级别 |
|------|---------|---------|
| 实时DAU | 下降 > 20% | 中 |
| 接口成功率 | < 99% | 高 |
| 支付成功率 | < 95% | 高 |

**监控工具**：Grafana + Prometheus AlertManager

### 7.2 定期分析报告
- **日报**：DAU、新增用户、GMV、转化漏斗（每天9点）
- **周报**：核心指标趋势、留存分析、功能使用分析（每周一）
- **月报**：月度总结、用户增长、收入分析、竞品对比（每月1号）
